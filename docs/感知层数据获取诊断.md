# 感知层数据获取失败诊断说明

## 问题现象

感知层输出 JSON 中大量字段为 `null`，且包含占位符 `{{current_time}}`，`flags` 显示 `weather_unavailable`。

## 根本原因分析

### 1. 核心问题：LLM 未实际调用工具

从日志可见，感知阶段只有一次 HTTP 请求到 Ollama，**没有出现「🔧 执行 X 个工具调用」**。说明大模型直接生成了 JSON 模板，而没有调用 `get_sensor_data`、`get_forecast_week`、`get_observe`、`get_crop_info` 等工具。

可能原因：
- **模型不支持或弱支持 function calling**：部分 Ollama 模型对工具调用支持较差，容易直接输出文本
- **工具未正确传入**：若 MCP 服务未连接，`tools` 列表为空，模型会退化为普通文本生成

### 2. 天气工具必须传 city 参数

`get_forecast_week` 和 `get_observe` 均需要 `city` 参数。用户问「曲靖会下雨吗」时，应调用 `get_forecast_week(city="Qujing")`。若未传 city 或 city 错误，接口会返回 error。

### 3. 作物与传感器字段映射

- **get_crop_info** 返回：`ideal_soil_moisture`、`min_temperature`、`max_temperature`
- 感知层输出期望：`ideal_moisture`、`min_temp`、`max_temp`（需做字段映射）

- **get_sensor_data** 返回：`temperature`、`humidity`、`soil_moisture`（无 `soil_temperature`）
- 感知层：`soil_temperature` 无数据时可填 `null`

## 已做修复

1. **Prompt 优化**（`prompts.py`）
   - 明确天气工具必须传 `city` 参数
   - 补充城市提取规则（曲靖→Qujing 等）
   - 说明作物与传感器字段映射关系

2. **客户端日志**（`client.py`）
   - 感知阶段打印 `感知层可用工具数: X -> [工具名列表]`
   - 若工具数为 0，输出警告并提示运行 `--init-only` 检查连接

3. **用户消息增强**
   - 在用户消息中显式要求「先调用工具获取数据，再输出 JSON」

## 建议排查步骤

### 步骤 1：确认 MCP 服务与工具加载

```bash
cd client/mcp-client
python client.py --init-only
```

预期：`7/7 个服务连接成功`，`19 个工具可用`。若失败，检查 `mcp_servers.json` 和各服务启动环境（如 uv、依赖）。

### 步骤 2：确认 Ollama 模型支持工具调用

建议使用对 function calling 支持较好的模型，例如：

```bash
ollama pull qwen2.5:7b      # 推荐
ollama pull llama3.1:8b     # 或
ollama pull mistral:7b      # 或
```

在 `.env` 中设置：
```
MODEL=qwen2.5:7b
```

### 步骤 3：查看感知层工具数量

运行客户端并提问后，检查日志中的 `感知层可用工具数`。若为 0，说明 sensor/weather/crop_knowledge 服务未正确加载。

### 步骤 4：手动验证各工具

在 Python 中可模拟调用：

```python
# 传感器（需先启动 MCP 服务）
# get_sensor_data(device_id="ESP8266_001") 应返回 demo 数据

# 天气（需要 OWM API Key 或内置 key 有效）
# get_forecast_week(city="Qujing") 
# get_observe(city="Qujing")

# 作物
# get_crop_info(crop_name="wheat") 应返回 ideal_soil_moisture 等
```

## 环境变量

- `OWM_API_KEY`：OpenWeatherMap API Key（天气服务）
- `BASE_URL`：Ollama 地址，默认 `http://localhost:11434/v1`
- `MODEL`：使用的模型名
