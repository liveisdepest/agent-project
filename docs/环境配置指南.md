# 环境配置指南

## 📋 前置要求

### 1. Python 环境
- Python 3.10 或更高版本
- 推荐使用 Python 3.11

### 2. uv 包管理器
```bash
# Windows (PowerShell)
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"

# macOS/Linux
curl -LsSf https://astral.sh/uv/install.sh | sh
```

### 3. LLM 服务

#### 选项 A: Ollama（推荐本地开发）
```bash
# 下载并安装 Ollama
# 访问 https://ollama.ai/download

# 拉取模型
ollama pull qwen2.5-coder
# 或
ollama pull deepseek-coder
```

#### 选项 B: OpenAI API
需要有效的 OpenAI API Key

#### 选项 C: 其他兼容服务
任何支持 OpenAI API 格式的服务

## 🔧 配置步骤

### 1. 配置客户端环境变量

```bash
cd client/mcp-client

# 复制示例配置文件
cp .env.example .env

# 编辑 .env 文件
# Windows: notepad .env
# macOS/Linux: nano .env
```

**配置说明**：

```env
# LLM 配置
API_KEY=ollama                          # Ollama 本地部署填 "ollama"
BASE_URL=http://localhost:11434/v1      # Ollama 默认地址
MODEL=qwen2.5-coder                     # 使用的模型名称

# 天气服务配置
OWM_API_KEY=your_key_here               # OpenWeatherMap API Key
```

### 2. 获取 OpenWeatherMap API Key

1. 访问 [OpenWeatherMap](https://openweathermap.org/api)
2. 点击 "Sign Up" 注册账号
3. 登录后进入 "API keys" 页面
4. 复制默认生成的 API Key 或创建新的
5. 将 API Key 填入 `.env` 文件的 `OWM_API_KEY`

**免费版限制**：
- 60 次/分钟
- 1,000 次/天
- 足够开发和测试使用

### 3. 安装依赖

```bash
# 在客户端目录
cd client/mcp-client
uv sync

# 或者使用 pip
pip install -r requirements.txt
```

### 4. 验证配置

```bash
# 测试连接（仅初始化，不启动聊天）
uv run client.py --init-only
```

**预期输出**：
```
开始连接 7 个MCP服务...
✅ 成功连接: sensor
✅ 成功连接: weather
✅ 成功连接: crop_knowledge
```

## 🚀 启动服务

### 方式一：命令行客户端（推荐测试）

```bash
cd client/mcp-client
uv run client.py
```

### 方式二：Web 服务（推荐生产）

```bash
# 启动 Web 服务器
cd server/web
uv run app.py

# 访问 http://localhost:8080
```

### 方式三：Arduino 硬件集成

1. 修改 `arduino_code.ino` 中的 WiFi 配置：
```cpp
const char* ssid = "你的WiFi名称";
const char* password = "你的WiFi密码";
const char* serverURL = "http://你的电脑IP:8080";
```

2. 上传代码到 ESP8266
3. 启动 Web 服务器
4. Arduino 会自动连接并上报数据

## 🔍 故障排查

### 问题 1: MCP 服务连接失败

**症状**：
```
❌ sensor 连接失败
```

**解决方案**：
1. 检查 Python 版本：`python --version`（需要 ≥3.10）
2. 检查 uv 安装：`uv --version`
3. 检查依赖安装：`cd server/sensor && uv sync`
4. 查看详细日志：修改 `client.py` 中的日志级别为 DEBUG

### 问题 2: 天气服务返回错误

**症状**：
```
无法获取 Qujing 的天气预报
```

**解决方案**：
1. 验证 API Key 是否正确
2. 检查网络连接
3. 确认 API Key 已激活（新注册需要等待几分钟）
4. 检查是否超过免费额度

### 问题 3: Ollama 连接失败

**症状**：
```
Connection refused to localhost:11434
```

**解决方案**：
1. 确认 Ollama 已启动：`ollama list`
2. 检查端口占用：`netstat -an | findstr 11434`
3. 重启 Ollama 服务

### 问题 4: 模型未找到

**症状**：
```
Model 'qwen2.5-coder' not found
```

**解决方案**：
```bash
# 拉取模型
ollama pull qwen2.5-coder

# 查看已安装的模型
ollama list
```

## 🔐 安全建议

1. **不要提交 .env 文件到 Git**
   - 已在 `.gitignore` 中配置
   - 包含敏感信息（API Keys）

2. **定期更换 API Keys**
   - 特别是在公开仓库中

3. **使用环境变量**
   - 生产环境建议使用系统环境变量
   - 避免硬编码敏感信息
