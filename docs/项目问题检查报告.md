# 项目问题检查报告

## 🔴 高优先级

### 1. 安全：天气 API Key 硬编码
**位置**：`server/weather/weather.py` 第 24-26 行

```python
if not api_key:
    return "137d6f6f171fc0f2f19011d3134a1501"  # 硬编码 key 泄露风险
```

**风险**：API Key 泄露到代码仓库，可能被滥用、产生费用。  
**建议**：移除 fallback，当 `OWM_API_KEY` 未设置时直接返回错误或空数据。  
**已修复**：已移除硬编码 key，未配置时天气接口返回空数据。

---

### 2. make_irrigation_decision 对 null 处理不足
**位置**：`server/decision/decision.py`

**问题**：`current_temperature` 等为必填 `float`，感知层 `air_temperature: null` 时，LLM 若传 null 会触发调用失败。  
**建议**：支持 `Optional[float]`，null 时使用合理默认值（如 25.0）。

---

### 3. 执行层 start_irrigation 缺少 zone 参数
**位置**：`server/irrigation/tools/start_irrigation.py`

**问题**：工具签名为 `start_irrigation(zone: str, duration_minutes: int)`，但决策输出仅有 `irrigation_duration_min`，无 `zone`。  
**建议**：在 ACTION_PROMPT 中明确说明使用默认 zone（如 `"zone_1"` 或 `"A"`），或扩展决策输出包含 zone。

---

## 🟡 中优先级

### 4. load_dotenv 依赖启动目录
**位置**：`client/mcp-client/client.py` 第 25 行

**问题**：`load_dotenv()` 默认从当前工作目录加载 `.env`，从项目根目录运行脚本时可能加载失败。  
**建议**：显式指定路径：`load_dotenv(os.path.join(os.path.dirname(__file__), '.env'))`

---

### 5. mcp_servers.json 使用相对路径
**位置**：`client/mcp-client/mcp_servers.json`

**问题**：`cwd` 使用 `"..\\..\\server\\weather"`，依赖客户端启动目录。从其他目录启动时可能出错。  
**建议**：在 client 中把 cwd 解析为基于配置文件的绝对路径。

---

### 6. DEBUG 输出应使用 logger
**位置**：`client/mcp-client/client.py` 第 99、115、125 行

```python
print(f"DEBUG: Attempting to connect to {server_id}...")
```

**建议**：改为 `logger.debug(...)`，便于统一控制日志级别。

---

### 7. 灌溉 API 依赖外部服务
**位置**：`server/irrigation/config.yaml`、`actuators/pump.py`

**问题**：`api_url: "http://127.0.0.1:8080/api"`，若 8080 无服务，`start_irrigation` 会失败。  
**建议**：文档中说明需启动对应硬件 API；或增加 mock 模式，便于本地开发。

---
